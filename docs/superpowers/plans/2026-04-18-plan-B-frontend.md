## Design discipline (first-class quality gate)

These skills MUST be consulted at hours 6, 18, 30. Frontend PRs without a design review get rejected:

- **`/ui-ux-pro-max`** — layout hierarchy, whitespace discipline
- **`taste`** — color, typography, visual voice, kills AI-slop defaults
- **`impeccable`** — pixel alignment, micro-spacing, state coverage (hover/focus/active/disabled/loading/empty/error)
- **`emilkowalski-animation`** — spring physics, motion feel

**Forbidden (per PRD §7.2.2):**
- Purple-to-pink gradients
- Generic glassmorphism
- Emoji in headers or nav
- Default shadcn card + default button stacks
- Drop shadows on flat elements
- AI badges / sparkle icons / robot avatars
- Decorative 3D illustrations
- Lorem ipsum or "Your content here"
- Spinner loading states (use skeleton screens that match final layout exactly)

**Required:**
- All numeric cells have `.tnum` class
- IDs render in JetBrains Mono
- Motion: 120–320ms durations only; spring physics (`stiffness: 260, damping: 26`) for entrances; `ease-out` for exits
- `prefers-reduced-motion`: animations become instant (already wired in `globals.css`)
- Every screen has: empty state, error state, loading skeleton, hover, focus, disabled

## Escalation signals

- Animation <60fps on demo laptop → profile, reduce to essential motion, ship
- Map lag or tile-load failures → switch to static SVG overlay, don't fight Leaflet at hour 30
- shadcn defaults leaking through ("looks AI-generated") → design review skills immediately, rip it out
- WS disconnect during live demo → surface via status dot; don't silently degrade
